# 整理について

25H2実装について整理します。

## 整理のポイント: シンプル・軽量に
- 似たものの排除(一つのことは一箇所に)
  - APIのコピー記述(インポート)はせず、概要をdocに書くことで、見通しをよくする
  - APIのインポート記述には以下の問題あり
    - ネストが深くなり、重要なポイントが見にくい
    - 全体量が多くなり、全体の見通しが悪い
    - テクニカルに、インポートによる重要情報の欠落、同期忘れの問題
- 読みやすく
  - 章立てをシンプルに。
  - 一般的なこと(プロトコルやライブラリ仕様など)は、削除し、参考情報へのリンクとする。できれば、なぜ・どう取り入れたいかを書く。
  - 絵よりも表や箇条書き。docusaurusへ絵のインライン表示は、拡大縮小に問題があり、重要なことが読み取れない。
- 通信制御の記述やシーケン図などは実体に合わないので、ストリーム転送の仕組みを反映
